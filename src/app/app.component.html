DAXTA Technical Assessment: Phase I

Define Calculated$

Given two observables NestedArray$ and ReductionOperator$, create a new observable Calculated$ that applies the reduction operator to a flattened version of NestedArray$.

  • A well formed element of NestedArray$ will either be an integer or another NestedArray$. There are no limits to the depth of the array.
  • ReductionOperator$ should emit either "Add" or "Multiply".
  • When "Add" is emitted, Calculated$ should be set to the sum of all of the elements of the flattened array. When "Multiply" is emitted, Calculated$ should be set to the product of all of the elements of the flattened array.
  • Example: Given [2, 3, 4, [5, [6, 7, 8], 9]] and "Add" the output should be calculated by the operation 2+3+4+5+6+7+8+9 leading to the solution of 54.

Please provide relevant tests to accompany your product.

Note: A new solution should be calculated/emitted whenever either input stream is altered. If an input is changed while the task is still calculating, the result from the first task should emit before the second.
